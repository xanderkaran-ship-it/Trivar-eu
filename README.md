## Quick start — upload to GitHub and go live

### Step 1 — Upload to GitHub

1. Go to **github.com** → **New repository**
2. Name it `trivar` · Visibility: **Public** · No README/gitignore
3. Click **Create repository**
4. On the empty repo page click **uploading an existing file**
5. Select **all files inside this zip** (including hidden `.github/` and `admin/` folders) and drag them in
6. Commit message: `Initial deploy` → **Commit changes**

### Step 2 — Enable GitHub Pages

1. Repo → **Settings** → **Pages** (left sidebar)
2. Source: **GitHub Actions** *(not "Deploy from branch")*
3. Click **Save** → go to **Actions** tab → watch the deploy (~60 s)
4. Site is live at `https://<your-username>.github.io/trivar/`

### Step 3 — Connect trivar.eu

## Decap CMS — editing content without code

Admin panel: **trivar.eu/admin/**

### One-time setup (15 min)

Decap needs a GitHub OAuth proxy. Easiest option:

1. Create free account at [app.netlify.com](https://app.netlify.com)
2. **New site → Import from GitHub** → pick your `trivar` repo
3. **Site settings → Identity → Enable Identity**
4. **Identity → Registration → Invite only**
5. **Identity → External providers → GitHub** → enable
6. Copy your Netlify site URL (e.g. `https://trivar-cms.netlify.app`)
7. Open `admin/config.yml`, replace the `base_url` line:
   ```yaml
   base_url: https://trivar-cms.netlify.app
   ```
8. Also update the `repo:` line with your GitHub username:
   ```yaml
   repo: YOUR_GITHUB_USERNAME/trivar
   ```
9. Commit and push → done.

### What you can edit in the CMS

| Section | What it does |
|---------|-------------|
| 🎯 Job Listings | Add/edit/remove jobs on the Karriere page. Toggle `active` to hide a role instantly |
| ⚙️ Site Settings | Company name, address, phone, email |
| 🖼️ Media Library | Upload images for job descriptions |

**Save cycle:** Edit → Save (creates draft) → Publish (merges to main) → GitHub Actions redeploys in ~60 s.

## Contact form — making it actually send email

Edit `js/contact-form.js` line 12 once you have a Worker URL (see below).

### Deploy the contact Worker (10 min, free)

1. Get a free API key from [resend.com](https://resend.com) (3,000 emails/month free)
2. Verify `trivar.eu` in Resend (add a DNS TXT record they give you)

```bash
cd api/
npm install -g wrangler
wrangler login
wrangler secret put RESEND_API_KEY    # paste Resend key
wrangler deploy
# outputs: https://trivar-contact.xxx.workers.dev
```

3. Open `js/contact-form.js` and replace:
   ```js
   'https://trivar-contact.YOUR-CF-SUBDOMAIN.workers.dev'
   // with your actual Worker URL
   ```
4. Commit and push. Done — submissions arrive at Contact@trivar.eu.
